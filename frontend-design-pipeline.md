# Frontend design pipeline

Цель: сделать MVP визуально живее без изменения backend/API/pipeline-логики.

## 1. Upload UX

- Drag & drop должен иметь явное active-состояние.
- После выбора файла показывать карточку файла: имя, размер, тип.
- Дать явные действия: выбрать/заменить файл, убрать файл, запустить обработку.
- Во время загрузки показывать progress и анимированное состояние.

## 2. Pipeline/loading UX

- Добавить infinity loader в брендовых цветах.
- Сделать progress bar с shimmer-анимацией.
- Показать текущий этап с коротким описанием, что сейчас происходит.
- Сделать шаги pipeline визуально разделёнными на pending / active / done / failed.

## 3. Result loading states

- Пока грузятся summary/player/charts/objects, показывать skeleton states вместо пустых блоков.
- Ошибки не должны ломать весь экран результата.

## 4. Result actions

- На странице результата нужны быстрые действия:
  - загрузить новое видео;
  - вернуться в историю;
  - скопировать ссылку на результат.

## 5. Micro-interactions

- Hover lift/glow для карточек, кнопок, объектов и графиков.
- Плавное появление страниц/панелей.
- Активные элементы должны иметь короткую, но заметную анимацию.

## 6. Empty states

- Пустая история должна вести пользователя к загрузке видео.
- Пустые/ожидающие блоки должны выглядеть как состояние приложения, а не как отсутствие данных.

## Acceptance checklist

- `pnpm lint` проходит.
- `pnpm build` проходит.
- Изменения только на фронте и в этом документе.
- Backend, worker, Docker, БД, MinIO не меняются.
